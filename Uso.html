¿Que es Docker Compose?

Docker Compose es una herramienta versátil que te permite definir y gestionar aplicaciones multi-contenedor de forma sencilla. Con Docker Compose, puedes describir la configuración de tu entorno de desarrollo en un archivo YAML, especificando los servicios, volúmenes y redes necesarios para tu aplicación. Luego, con un solo comando, puedes crear y ejecutar todos los contenedores definidos en tu archivo de configuración


Caracteristicas Principales

  • Definición en un solo archivo: Docker Compose utiliza un archivo llamado docker-compose.yml en el que defines los servicios, redes y volúmenes que tu aplicación necesita. Todo está centralizado y es fácil de gestionar.
  • Comandos sencillos: Con Docker Compose, puedes ejecutar, detener y gestionar contenedores de manera sencilla usando comandos como docker-compose up para iniciar la aplicación, y docker-compose down para detenerla.
  • Compatibilidad con múltiples entornos: Docker Compose facilita la creación de entornos de desarrollo que se pueden replicar en producción. Puedes definir diferentes configuraciones según el entorno (desarrollo, prueba, producción).
  • Ejecución paralela de contenedores: Compose te permite ejecutar todos los contenedores definidos en el archivo docker-compose.yml en paralelo, gestionando las dependencias y el orden en que deben iniciarse los servicios.

Comandos de Docker

Comando Explicacion
docker run [opciones] [imagen] [comando] Crea y ejecuta contenedores a partir de imágenes de Docker.
docker pull/push [nombre_imagen] Los comandos docker pull y docker push se utilizan para descargar y subir imágenes a un registro de Docker, como Docker Hub.
docker images [opciones] Se utiliza para mostrar una lista de las imágenes disponibles en el sistema.
docker ps[opciones] Se utiliza para mostrar una lista de los contenedores en ejecución.
docker stop [opciones] [nombre_contenedor] Se utiliza para detener uno o varios contenedores en ejecución.
docker rm [opciones] [nombre_contenedor] Se utiliza para eliminar uno o varios contenedores.
docker network [opciones] Se utiliza para gestionar las redes en Docker. Proporciona una forma de crear y configurar redes virtuales para permitir la comunicación entre contenedores.
docker volume [opciones] [subcomando] Se utiliza para gestionar los volúmenes en Docker. Los volúmenes son una forma de persistir datos y compartirlos entre los contenedores y el host.
docker-compose [opciones] [comando] Se utiliza para definir y gestionar aplicaciones multi-contenedor utilizando un archivo de configuración llamado docker-compose.yml. Este archivo describe los servicios, las redes y los volúmenes necesarios para la aplicación.
